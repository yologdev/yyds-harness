Objective:
Write a deterministic Python script (`scripts/synthesize_learnings.py`) that reads `memory/learnings.jsonl` and regenerates `memory/active_learnings.md` using time-weighted compression tiers. Run it to produce the first yyds-native active learnings context.

Why this matters:

With 255 total entries and 49 recent ones, yyds has substantial self-knowledge that is invisible to every session. The active learnings file is how past lessons influence current decisions — without it, each session starts from the same stale gen0 baseline. The LLM-based synthesize workflow is failing silently; a deterministic script removes the API dependency and makes synthesis reproducible.

Implementation Notes:

The script should use these time-weighted compression tiers (from synthesize.yml):

- **Recent (last 2 weeks):** Entries with `day >= latest_day - 14`. Render each as:
  ```
  ### Lesson: {title}
  **Day:** {day} | **Date:** {date} | **Source:** {source}
  **Context:** {context}
  {takeaway}
  ```
  (blank line between entries)

- **Medium (2-8 weeks):** Entries with `latest_day - 56 <= day < latest_day - 14`. Condense each to:
  ```
  **{title}** (Day {day}): {condensed takeaway, first 1-2 sentences}
  ```
  Drop the full context; keep only the insight.

- **Old (8+ weeks):** Entries with `day < latest_day - 56`. Group by theme:
  Use `pattern_key` field if present (e.g., `tests.add_before_change` → theme "Testing") or derive theme from title keywords. Group into `### {Theme}` subsections with 2-3 sentence summaries per group. List up to 5 themes max; merge sparse groups into a catch-all "Early Learning Patterns" theme.
